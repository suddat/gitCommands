• 1st Part Basic Commands

  • git commit : - exactly like svn. (svn commit -m)
  • git add filename/filenames : - to add file to repository.
  • git diff : to find difference between two versions of file.
  • git checkout HEAD filename : Discards changes in the working directory. (rm filename && svn up)
  • git reset HEAD filename : Unstages file changes in the staging area.
  • git reset SHA : Can be used to reset to a previous commit in your commit history. (svn -r revision no)

  • 2nd Part will be about Branch, Merge and TeamWork Commands

  • git branch : shows the branch of repository. (branches are nothing but a completed version, to add on something new on the completed version create a branch, such that if anything goes wrong on the new version can be reverted back to the main branch the completed version will be called as Master branch.)
  • git checkout branch_name : with 'checkout' can switch to new branch_name and it wont affect the master now, both have the same commit code, but post switching the git checkout, master will remain unaffected.
  • git merge branch_name : merge from new version to master version because branch_name has the latest code.
  • git checkout HEAD filename : Discards changes in the working directory. (rm filename && svn up)
  • git branch -d branch_name : now the changes has been merged into master, purpose of fencing is done, so delete it with this command.
  • git clone: Creates a local copy of a remote.
  • git remote -v: Lists a Git project's remotes.
  • git fetch: Fetches work from the remote into the local copy.
  • git merge origin/master: Merges origin/master into your local branch.
  • git push origin : Pushes a local branch to the origin remote.
